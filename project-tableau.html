11 — data & visualisation

Data Viz Tableau
Olist Brasil

tableau desktop data visualisation e-commerce dashboard
Voir le dashboard ↗
Voir sur Tableau Public ↗
Contexte

Dataset public Olist — 99 441 commandes e-commerce brésiliennes réparties sur 3 tables CSV (orders, customers, payments), couvrant la période 2016-2018.

Objectif

Concevoir un dashboard analytique complet sur Tableau Desktop — jointure multi-tables, création de champs calculés, production de 8 vues distinctes (cartes, séries temporelles, bar charts) consolidées avec filtres liés cross-vues.

Résultat

Dashboard interactif en 8 vues avec filtres liés par période, état et type de paiement. Insights clés : São Paulo = 42% des commandes, 97% livrées, pic Black Friday nov. 2017 (7 863 commandes), revenu total R$ 16 008 872.

stack technique
Tableau Desktop CSV multi-tables Champs calculés DATEDIFF Carte choroplèthe Filtres liés
8 vues produites
01 Carte — Nombre de commandes par état brésilien
02 Bar chart — Commandes par statut (delivered, shipped, canceled…)
03 Bar chart — Modes de paiement (credit card, boleto, voucher…)
04 Bar chart — Montant moyen par type de paiement (en R$)
05 Série temporelle — Évolution mensuelle des commandes 2016-2018
06 Bar chart — Top 10 villes avec le plus de commandes
07 Graphique lignes — Délai de livraison réel vs estimé par état
08 Carte choroplèthe — Valeur totale des paiements par état
